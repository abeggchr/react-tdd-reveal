What is TDD?

Three laws of TDD

  1. You are not allowed to write any production code unless it is to make a failing unit test pass.
  2. You are not allowed to write any more of a unit test than is sufficient to fail; and compilation failures are failures.
  3. You are not allowed to write any more production code than is sufficient to pass the one failing unit test.

What's the benefit of TDD?

  • Your brain's reward system is continuously pleased!
    • Short feedback cycle / Smaller pieces
  • Refactor with confidence
  • Clean code that works
  • Drive design
  • Stops writing what we dont need
  • Less debugging

TDD needs practice

Writing tests fluently, knowing frameworks etc.

Writing tests that encourage to do refactorings, not prevent refactorings.
